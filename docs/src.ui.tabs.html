src.ui.tabs.boot_flow_tab module

class src.ui.tabs.boot_flow_tab.BootFlowTab(master: CTkFrame, boot_flow: BootFlow, app)[source]

Bases: BaseTab

A class representing the BootFlow tab, inheriting from BaseTab. This tab provides functionality for managing processors within the boot flow, and sceduling the different boot events.

add_processor()[source]

Adds a new processor to the BootFlow instance based on user input from the popup window. The new processor is also added to the BootFlow window’s treeview, and the popup is then closed.

add_processor_popUp()[source]

Opens a popup window that allows the user to add a new processor to the BootFlow. The popup contains input fields for the processor name and an optional comment.

create_widgets()[source]

Creates and configures the widgets for the tab, including the BootFlow window, navigation buttons (up, plus, down), and sets up the grid configuration for layout management.

on_tab_selected(event)[source]

Handles the event when the tab is selected. This method triggers the update of the listbox to ensure it displays the most recent data.

Parameters:

eventEvent

The event object associated with the tab selection.

update_listbox()[source]

Updates the listbox (treeview) in the BootFlow window with the current processors. This method refreshes the displayed list to reflect any changes made to the BootFlow.

Bases: BaseTab

A class representing the initialization tab, inheriting from BaseTab. It allows users to add, update, or delete initialization events in the boot flow.

add_or_update_event()[source]

Adds a new event or updates an existing one in the BootFlow instance based on the current values in the input fields. The method handles different event types (None, Trigger, Wait) and updates the corresponding processor and event attributes.

clear_selection(event)[source]

Clears the current selection in the treeview, resets the input fields, and clears the comment box when triggered by a mouse click or escape key.

Parameters:

eventEvent

The event object associated with the action.

create_widgets()[source]

Creates and configures the widgets for the tab, including input fields, buttons, comment field, and a treeview for displaying events. It also sets up grid configuration for proper layout management.

delete_event()[source]

Deletes the selected event(s) from the BootFlow instance and updates the treeview accordingly.

Returns:

list

A list of trigger names, or [“None”] if no triggers are available.

Parameters:

eventEvent

The event object associated with the treeview selection.

update_combobox()[source]

Updates the comboboxes in the input frame with the current list of processors and triggers from the BootFlow instance.

update_listbox()[source]

Updates the treeview with the current list of events from all processors in the BootFlow instance. The treeview displays the event name, duration, processor name, and whether the event is a trigger or wait event.
